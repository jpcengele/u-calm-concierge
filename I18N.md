# i18n workflow — propagate-locales

**TL;DR**: English is source of truth. Run `npm run i18n:sync` after any edit
to `en.json` or after adding new `t()` keys. de/fr/it auto-populate via
DeepL in ~10 seconds, except for voice-critical strings in
`i18n-overrides.json` which stay hand-written.

---

## How this works

Three layers:

| Layer | File | What it holds |
|---|---|---|
| **Source** | `src/i18n/locales/en.json` | English — authoritative |
| **Engine** | `scripts/sync-locales.py` | DeepL-powered translator + cache + validator |
| **Overrides** | `i18n-overrides.json` | Hand-written brand-voice translations that bypass DeepL |

When the script runs, for each non-English string it checks:
1. Is this key in `i18n-overrides.json` for the target locale? → use override verbatim.
2. Is the source text already cached for this locale? → use cached translation (free, instant).
3. Otherwise → call DeepL, store result in cache for next time.

Proper nouns (U-CALM, Lugano, Consider it done., etc.) are automatically
preserved via DeepL's `<x>`-tag mechanism — never translated even if
they're in the middle of a sentence.

## First-time setup (once per repo)

```
pip install -r scripts/requirements.txt
cp env.local.example .env.local
```

Edit `.env.local` — replace `your-deepl-api-key-here` with a real key.
Get a free key at <https://www.deepl.com/pro-api> (free tier: 500,000
characters/month, plenty for marketing sites).

Add `.env.local` and `.i18n-cache.json` to `.gitignore` if they aren't
already. Commit `i18n-overrides.json` — it's part of the brand voice.

Optionally wire into `package.json`:

```json
{
  "scripts": {
    "i18n:sync": "python3 scripts/sync-locales.py"
  }
}
```

## Day-to-day workflow

**Adding new copy:**
1. Edit a component — use `{t("some.new.key")}` in JSX.
2. Add the English string to `src/i18n/locales/en.json` at that key path.
3. Run `npm run i18n:sync`.
4. Commit all four locale files + any updated cache + overrides.

**Editing existing copy:**
1. Change the string in `en.json`.
2. Run `npm run i18n:sync` — only that string gets re-translated (cache sees source changed).
3. Commit.

**Adding a new locale:**
1. Create `src/i18n/locales/<code>.json` with `{}` placeholder.
2. Add the locale code to `TARGET_LOCALES` and `DEEPL_TARGETS` in `sync-locales.py`.
3. Run `npm run i18n:sync`.

**When DeepL's translation loses the brand voice:**
1. Inspect the new locale file, spot the offending string.
2. Open `i18n-overrides.json`, add the hand-written version under
   `"de.key.path.here": "..."` (and likewise for `fr`, `it`).
3. Re-run the sync — the override wins, DeepL never touches that key again.

---

## What the overrides file looks like

```json
{
  "de": {
    "brand.strapline": "Erledigt.",
    "home.signature.headline": "…crafted German that preserves brand voice…"
  },
  "fr": {
    "brand.strapline": "Considérez que c'est fait."
  }
}
```

Not every key needs an override — only the ones where machine translation
produces flat or literal output. Straplines, hero headlines, and poetic
signature hooks usually need overrides. Navigation labels, CTAs, and
simple body copy usually don't.

## CLI reference

```
python3 scripts/sync-locales.py                # sync all target locales
python3 scripts/sync-locales.py --locale de    # sync one
python3 scripts/sync-locales.py --dry-run      # show what would change
python3 scripts/sync-locales.py --clear-cache  # invalidate translation cache
```

## Do-not-translate list

`sync-locales.py` contains a `DO_NOT_TRANSLATE` list of proper nouns and
fixed phrases that must stay in English across every locale (brand names,
product names, city names, the U-CALM acronym expansion, etc.). Extend
this list per brand if needed — e.g., Ascent Aviation would add
`"Ascent Aviation Advisors"`, `"Triple A all the way."`,
`"London"`, etc.

---

## Cache hygiene

The cache (`.i18n-cache.json`) lives at the repo root. It's an optimisation
— you can delete it any time and everything still works (every string
gets re-translated on the next sync, hitting the DeepL quota). Commit it
or don't; behaviour is the same either way. Committing makes CI runs
faster and cheaper.

## Why this design

- **Source of truth is English, not a CMS or spreadsheet.** Every edit
  happens in one place. The locales derive; they are not authored
  independently. No drift possible.
- **DeepL handles 80% automatically.** UI labels, CTAs, most body copy
  are fine on first pass.
- **Overrides handle the 20% that needs voice.** Straplines, headlines,
  poetic lines live in one short file. Reviewable at a glance.
- **Cache makes re-runs free.** Add one new string → one DeepL call; not
  100 calls for the whole site. Scales to any site size.
- **Script travels with the repo.** No external service, no SaaS lock-in.
  If DeepL changes, swap the backend (Google Translate, Claude API,
  whatever) — the surrounding workflow is unchanged.

---

## Adapting to another brand

- Change the `DO_NOT_TRANSLATE` list in `sync-locales.py` to match the
  new brand (names, cities, straplines that don't translate).
- Overwrite `i18n-overrides.json` with the new brand's voice-critical
  strings (mostly the first sync will show you which ones need them).
- Everything else is identical. Same script, same workflow.
